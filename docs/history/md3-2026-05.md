# Historical Reference — Material Design 3 Token Dump (2026-05)

## Background

This file preserves the dark-variant color token set from Startino's
pre-Catppuccin color system, generated via the
[Material Theme Builder](https://material-foundation.github.io/material-theme-builder/)
in May 2026.

The source colors fed into the MD3 generator were:

- **`#45dfa4`** — mint primary (survived into the new system as `primary` / `green`)
- **`#3cddc7`** — teal secondary (dropped; went unused in practice)
- **`#ffafd3`** — pink tertiary (promoted to Startino's `secondary` / `pink` in the new system)
- **`#ffb4ab`** — red error (remains the error semantic in both systems)
- **`#171919`** — background seed

The MD3 generator produced a 30-token surface palette. While the individual
accent colors were usable, the system as a whole did not yield the
perceptually-equal accent ramp that Catppuccin demonstrates — the "container"
token pairs in particular went almost entirely unused in day-to-day UI work,
and the tonal progression across roles was not intuitive when assembling
components. This dissatisfaction was the proximate cause of the migration.

The **light variant** was not preserved. If a future archaeologist needs it,
the Material Theme Builder at
<https://material-foundation.github.io/material-theme-builder/> can regenerate
it from the source colors listed above.

## Why We Migrated

Of the colors that came out of the MD3 system, the ones that survived into
day-to-day use were the mint primary and the pink tertiary — the latter
eventually becoming the new `secondary`. The teal secondary disappeared almost
immediately; it looked close enough to mint that the two conflated in
components where both appeared. The container token pairs (`primary-container`,
`on-primary-container`, etc.) are genuinely useful in Material-native UIs like
Android, but Startino's SvelteKit surfaces don't follow Material component
structure, so the tonal layers went entirely unused and just added cognitive
overhead. Catppuccin's named neutral stack (`crust` through `text`) plus a
curated set of 14 semantic accents maps more naturally to how we actually
build components, and the port ecosystem means theming external tools is a
mechanical substitution pass rather than a per-tool color derivation exercise.
