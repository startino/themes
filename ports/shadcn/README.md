# Startino Themes — shadcn port

Drop-in shadcn theme tokens for Startino's four palette flavors. Replaces the `:root` block in your project's `app.css`. Works for both **shadcn-svelte** and **shadcn-ui (React)** — both consume the same CSS custom properties.

## What this is

Each file exports the full set of CSS custom properties that shadcn components read (e.g. `--background`, `--primary`, `--ring`, `--sidebar-*`). The values use modern `oklch(...)` notation, consistent with Tailwind v4 and current shadcn conventions. No JavaScript, no build step.

Available flavors:

| File | Flavor | Mode |
|------|--------|------|
| `dist/startino-mercury.css` | Mercury | Light |
| `dist/startino-mars.css` | Mars | Dark |
| `dist/startino-jupiter.css` | Jupiter | Dark |
| `dist/startino-neptune.css` | Neptune | Dark |

## Install — single-file (recommended for theme switchers)

```css
/* app.css */
@import "@startino/themes/shadcn/dist/startino.css";
```

Then toggle `data-flavor` at runtime to switch themes:

```html
<html data-flavor="mars">
```

Available values: `mercury`, `mars`, `jupiter`, `neptune`. When no `data-flavor` attribute is set, Neptune is used as the default (it also sets `:root` directly).

## Install — per-flavor (build-time selection)

```css
/* app.css */
@import "@startino/themes/shadcn/dist/startino-neptune.css";
```

Picks one flavor at build time. Switch by changing the import.

## shadcn-svelte vs shadcn-ui

Both work — just paste the contents into your project's `app.css` after running `npx shadcn-svelte init` (or the React `npx shadcn init` equivalent). Remove or comment out the existing `:root { ... }` block that the init command generates before importing.

## Customisation

Each variable maps to a Catppuccin-equivalent palette role. If you want to change which role drives a given variable, edit your local copy of the file. The mapping is:

| shadcn variable | Startino role |
|----------------|---------------|
| `--background` | `base` |
| `--foreground` | `text` |
| `--card` | `mantle` |
| `--card-foreground` | `text` |
| `--popover` | `surface0` |
| `--popover-foreground` | `text` |
| `--primary` | `green` (mint) |
| `--primary-foreground` | `crust` |
| `--secondary` | `pink` |
| `--secondary-foreground` | `crust` |
| `--muted` | `surface1` |
| `--muted-foreground` | `subtext0` |
| `--accent` | `surface2` |
| `--accent-foreground` | `text` |
| `--destructive` | `red` |
| `--destructive-foreground` | `crust` |
| `--border` | `overlay0` |
| `--input` | `surface0` |
| `--ring` | `green` |
| `--chart-1..5` | green, pink, blue, peach, mauve |
| `--sidebar-*` | mirrors main roles above |
| `--radius` | `0.625rem` (literal) |

All files are generated by `prism` — do not edit the `dist/` files by hand. To regenerate:

```sh
cd /shared/themes
bun compiler/prism.ts --port shadcn
```
