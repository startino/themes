# Startino Themes — tmux

## What this is

Flavor files for the [Catppuccin tmux plugin](https://github.com/catppuccin/tmux) with Startino's brand color palette swapped in. The plugin's styling, modules, and separator logic are unchanged — only the `@thm_*` color variables are overridden.

Available in 4 planet flavors: Mercury (light), Mars, Jupiter, and Neptune (dark). Generated by the prism compiler from `tmux.njk`.

## Install

### 1. Install the Catppuccin tmux plugin via TPM

Add to `~/.tmux.conf`:

```tmux
set -g @plugin 'catppuccin/tmux'
```

Then install with `<prefix> + I` (TPM).

### 2. Drop the Startino flavor file into the plugin's themes directory

```sh
# For Neptune (dark) — change to mercury / mars / jupiter as needed
cp dist/catppuccin_neptune_tmux.conf \
  ~/.tmux/plugins/catppuccin/tmux/themes/catppuccin_neptune_tmux.conf
```

Or symlink so future `bun run build` runs update the live file automatically:

```sh
ln -sf "$(pwd)/dist/catppuccin_neptune_tmux.conf" \
  ~/.tmux/plugins/catppuccin/tmux/themes/catppuccin_neptune_tmux.conf
```

### 3. Set the flavor in `~/.tmux.conf`

```tmux
set -g @catppuccin_flavor "neptune"
```

### 4. Reload tmux

```sh
tmux source-file ~/.tmux.conf
```

## How it works

The Catppuccin plugin sources its flavor file at load time via:

```
source -F "#{d:current_file}/themes/catppuccin_#{@catppuccin_flavor}_tmux.conf"
```

All `@thm_*` color variables are then referenced lazily (`#{@thm_*}`) throughout the plugin's style strings. Our flavor file sets every one of those 30 variables to Startino colors — the plugin reads them at render time and never sees Catppuccin values.

TPM updates (`<prefix> + U`) don't touch arbitrary files placed in `themes/`, so a copied file survives updates. A symlink is even safer: it always points at the latest compiled output.

## Color variables emitted

All 30 variables Catppuccin's mocha flavor file declares:

`@thm_bg`, `@thm_fg`, `@thm_rosewater`, `@thm_flamingo`, `@thm_pink`,
`@thm_mauve`, `@thm_red`, `@thm_maroon`, `@thm_peach`, `@thm_yellow`,
`@thm_green`, `@thm_teal`, `@thm_sky`, `@thm_sapphire`, `@thm_blue`,
`@thm_lavender`, `@thm_subtext_1`, `@thm_subtext_0`, `@thm_overlay_2`,
`@thm_overlay_1`, `@thm_overlay_0`, `@thm_surface_2`, `@thm_surface_1`,
`@thm_surface_0`, `@thm_mantle`, `@thm_crust`.

No `@thm_primary` / `@thm_secondary` aliases are emitted — those were
standalone-port conventions and the Catppuccin plugin doesn't reference them.
