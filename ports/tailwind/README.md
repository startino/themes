# Startino Themes — Tailwind port

Pre-built CSS custom properties for all four Startino flavors (Mercury, Mars, Jupiter, Neptune), ready to drop into any Tailwind v4 project.

## Installation (Tailwind v4, CSS-first)

Install the package and import the generated stylesheet at the top of your CSS entry point:

```css
@import "@startino/themes/tailwind/dist/startino.css";
```

Neptune is the default flavor — active on `:root` with no `data-flavor` attribute required. To switch flavors, set `data-flavor` on `<html>` (or any ancestor element):

```html
<!-- Light theme -->
<html data-flavor="mercury">

<!-- Dark themes -->
<html data-flavor="mars">
<html data-flavor="jupiter">
<html data-flavor="neptune">  <!-- default if omitted -->
```

Once imported, every role is available as a Tailwind design token. Tailwind v4 automatically exposes CSS custom properties prefixed `--color-*` as color utilities:

```html
<div class="bg-base text-text border-surface-1">
  <button class="bg-primary text-base">Click me</button>
</div>
```

### Available tokens

| Token | Role |
|---|---|
| `--color-primary` | Green (brand primary) |
| `--color-secondary` | Pink (brand secondary) |
| `--color-error` | Red |
| `--color-warning` | Peach |
| `--color-info` | Blue |
| `--color-success` | Green |
| `--color-link` | Blue |
| `--color-base` … `--color-crust` | 12 neutral surface/text roles |
| `--color-rosewater` … `--color-lavender` | 14 accent roles |

## Tailwind v3 (config-driven)

This port targets Tailwind v4's CSS-first approach. For Tailwind v3, the generated stylesheet still works as plain CSS — reference the custom property names from `dist/startino.css` and wire them manually in your `tailwind.config.js`:

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary:   'var(--color-primary)',
        secondary: 'var(--color-secondary)',
        base:      'var(--color-base)',
        // … add the roles you need
      },
    },
  },
}
```

## Regenerating

The stylesheet is generated by [prism](../../compiler/prism.ts) — do not edit it by hand.

```sh
bun compiler/prism.ts --port tailwind        # regenerate
bun compiler/prism.ts --port tailwind --check # verify no drift
```
